Checkpoint、LoRA、VAE 到底是什麼？理解生成式 AI 模型

發表於2026-05-20|更新於2026-05-27|AI生成式 AI
|瀏覽量:

科技趣事

只需要查看你的點讚就能分析出你這人，有大神將自己的貼文給了 AI，最終 AI 分析出他是怎樣的人，結果卻很精確。

文章來源

前言

大約在六年前（2020 年左右），我第一次接觸了影像處理課程。那時候玩的是傳統的機器學習與深度學習，每天都在大費周章地自己用爬蟲圈資料、動手 Label。

從跑自己的貓狗辨識、手勢辨識，到後來的蔬菜辨識模型，後來出現遷移式學習 (Transfer Learning) 來訓練模型，當時最火紅的 Mask-RCNN 和 YOLOv4 影像物件辨識模型做 Fine-tuning。

到了 2024 年，我開始接觸 Stable Diffusion，轉向生成式 AI 的領域，用它來生成動漫圖片與插圖。不得不吐槽，Stable Diffusion 的 WebUI 介面的參數調校和設定常常讓人眼花撩亂。直到 2026 年初，我全面轉用節點式的 ComfyUI，一整個超舒爽…

不過，無論是 WebUI 還是 ComfyUI，最讓新手卡關的往往是：

❓ 「為什麼有這麼多種 Model 節點？Checkpoint、LoRA、VAE 到底是幹嘛的？」

本篇筆記不涉及太硬核的數學知識與過深原理。將以實務應用為主，徹底理清這些模型的底層行為與概念！

Model 到底是什麼？

簡單說，你丟東西進去，它輸出一個結果。訓練的時候它看過大量資料，學會了怎麼判斷、怎麼生成。訓練完之後，會存成一個檔案，也就是我們常說的 model 檔。

常見的格式長這樣

  • .pt / .pth：PyTorch 的格式。
  • .safetensors：現在 SD 生態系的主流格式，比較安全。

💡 結論： Model 檔就是裝著「訓練結果」的容器。

CV 時代玩過的那些 Model

在影像處理那個階段，接觸的 Model 大概分兩種路線：自己從頭訓練，跟拿別人的來微調（遷移學習）。

1. 自行訓練模型

  • 做法：從零開始收資料、標記（Label）、訓練。貓狗辨識、手勢辨識、蔬菜辨識都是這個路線。
  • 優缺點：優點是完全客製化；缺點是需要大量資料跟運算資源，在那時候 GPU 至少要 12GB VRAM。而且如果資料不夠，或者 Label 亂 Label，效果通常很普通或者很爛，準確率須達到 95 以上需要花點工夫，還要避免過擬合。

2. 遷移式學習 (Transfer Learning)

  • 做法：做遷移式學習的時候，通常是拿訓練好的權重，然後把最後幾層換成自己的分類層，再丟自己的資料去微調。
  • 優缺點：記憶體需求大、要跑一段時間，但準確率極高（85-95%）且又穩。最常見就是用 YOLOv4、faster-rcnn、mask-rcnn 來做效果最好。

Base Model / Checkpoint

到了生成式 AI 的時代，Stable Diffusion / ComfyUI 裡的 Model 變得更細、更模組化、更多樣職責。在 ComfyUI 的工作流裡，不同的 Model 負責不同的事情，組合起來才是完整的生成流程。

生成式會有個 Base Model 為基底，而 Stable Diffusion / ComfyUI 裡稱為 Checkpoint。這 Model 是 GB 起跳，訓練集非常大：

基礎模型 (Base Model) 優勢與特性 缺點與限制
SD 1.5 最經典長壽。資源需求極低，社群免費資源（LoRA、微調模型）最豐富。 原生解析度低（512x512），遠景人物或複雜細節容易崩壞。
SDXL 原生解析度提升至 1024x1024，畫面細節更好，光影材質與構圖大幅提升。 運算資源需求比 1.5 高，需要較高規格的顯示卡。
Flux 2024 年後崛起。生成品質極強，手部細節、文字生成與指令理解力大幅躍進。 檔案巨大，極度吃重電腦硬體資源與 VRAM 顯示記憶體。

💡 核心觀念：選 Base Model 就像選電腦主機板，其他外掛配件（LoRA、ControlNet）都必須跟著它走。版本不相容跑出來的圖片就會變爛圖，或者「生成結果異常」或「畫面崩壞」！

LoRA

要微調 Base Model 的模型，需要極大的 VRAM、極大時間以及昂貴的電腦。但 LoRA 採用了一個聰明的做法：

  • 凍結原模型：它不改變原本模型那龐大的參數。
  • 外掛小插槽：它只在模型的旁支加入一個非常小的矩陣（低秩矩陣）來訓練。
  • 輕量高效：最後訓練出來的 LoRA 檔案通常只有幾 MB，但能精準改變模型的輸出風格，例如讓 AI 畫出特定動漫角色，或讓語音模型用特定人聲。
  • 訓練簡易：訓練集與驗證集只需要少量資料，以圖像只需 五張圖片 就能跑出可用的 LoRA，非常厲害。

VAE

白話就是濾鏡，決定畫面的色彩、對比度、清晰度與去噪點，甚至五官與聲線的調整。

有些 Base Model 會內建，從字面上看職責為針對精度和細度，因此訓練需要非常大量的資料集且是高品質的，訓練時間也非常長。如果效果不好，會讓生成式出來的圖片不如預期。

VAE 差異 - 資料來源：aituts

ControlNet

如果圖像為人物，透過此 Model 控制骨架結構，強行規定人物的姿勢、手指、線稿輪廓必須順著你指定的線條走。

依據 ControlNet 骨架指定姿勢 - 資料來源：learnopencv

總結

到了 2026 年，生成式 AI 的生態系已經把這些組件模組化了。在 ComfyUI 中還能上網找尋 workflow.json 來直接完成配置與依賴進行生成，短短幾年進步速度如此之快！
