Python 3.15 將加入 Explicit Lazy Imports：lazy import 解決了什麼？

發表於2026-05-14|更新於2026-05-27|程式語言Python
|瀏覽量:

Python 3.15 將加入 lazy 新語法，非常實用，且解決 Python 2~3 時期以來常見的 import 問題。

一句話總結

透過新的 lazy 關鍵字，讓 import 延遲到第一次使用時才真正載入模組：

還沒有 Lazy Import 之前的痛點

Python 慣例是把所有 import 放在模組頂端，但這導致：

  • 啟動時觸發 import，載入大量可能用不到的模組
  • 命令列工具即使執行 --help，也要花數秒載入幾十個模組

目前的替代方案都有缺陷

塞進 function，是最常見、也最早被使用的方案。可以看到大多數 package source code 都會用這招，但缺點就是 import 到處放，難維護與控制。

方案 缺點
把 import 塞進函式（inline imports） import 會散落各處，依賴關係不清晰；標準庫已有約 17% 的 import 這樣做
importlib.util.LazyLoader 不支援 from ... import，需大量樣板程式碼
第三方套件（如 lazy_loader） 缺乏統一標準，各自為政
if TYPE_CHECKING: 只解決型別標注問題，不通用

設計哲學：四個核心原則

原則 說明
局部（Local） 惰性只影響標記了 lazy 的那一行，不遞迴影響被 import 的模組
顯式（Explicit） 名稱立刻綁定，但模組延遲到第一次使用才載入，清楚可見
受控（Controlled） 惰性由 import 方自己決定，不是外部強加
細粒度（Granular） 逐個 import 決定，而非全域開關

lazy 是一個軟關鍵字（soft keyword），只在 import 語句前有特殊意義，其他地方仍可作為普通識別符使用。

核心概念：Reification（實體化）

Lazy import 建立一個 proxy 物件，直到第一次使用時才真正載入模組，這個過程稱為 reification。
