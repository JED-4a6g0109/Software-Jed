何謂JWT?

發表於2024-08-12|更新於2026-05-18|APIJWT
|瀏覽量:

何謂JWT?

JWT 基於 JOSE 標準，以下技術來處理Token的簽名和加密，因為stateless特性，常出現 Web應用的用戶身份驗證與微服務之間的認證場景使用。

JOSE（JSON Object Signing and Encryption）

  • JWS(JSON Web Signature) RFC 7515
  • JWE(JSON Web Encryption) RFC 7516
  • JWK(JSON Web Key) RFC 7517
  • JWA(JSON Web Algorithms) RFC 751

JWT(Json Web Token)主要由三個部分組成，並用(.逗點)區隔這三個部分

  • Header
  • Payload
  • Verify Signature

Header

Header又分為兩個部分組成

  • Token Type
    ex: jwt
  • Algorithm
    ex: HMAC SHA256、RSA

JSON透過會被Base64Url編碼，形成JWT中的Header。

💡Base64Url
專為URL和檔案名稱的安全性設計，避免了特殊字元在傳輸過程中造成的相容性問題

💡為何不用base64?
在標準的 Base64 編碼在某些情況下可能不適合，因為它使用的字元集包括 +、/ 和 =，這些字元在 URL 中具有特殊意義，或在檔案名稱中不允許使用，可以自行在資料夾命名來驗證

Payload

包含關於用戶或其他數據的聲明（Claims）。Claims可以包括身份驗證信息、授權信息或其他數據。而Claims又分為

  • Registered Claims
  • Public Claims
  • Private Claims

Registered Claims

標準化claims，沒有強制加入claims，共有七個Claims組成，用意在於它們是標準化的，這些標準聲明的設計目的是為了確保不同系統或應用程序之間能夠互相理解和使用這些資料，最常見應該是iss、iat、exp、sub。

  • iss：發行JWT的實體。
  • sub：JWT所針對的用戶或主體。
  • aud：JWT的預期接收者。
  • exp：JWT過期的時間。
  • nbf：JWT在此時間之前無效。
  • iat：JWT發行的時間。
  • jti：JWT的唯一標識符，用於防止重放攻擊。

payload

簡單來說舉個例子，如果A系統發送了一個JWT，並使用了這些標準聲明，B系統接收到這個JWT時，因為這些聲明是標準化的，B系統可以輕鬆理解JWT中包含的資料，並正確地處理它。

⛔payload不要放入機密資訊，例如:密碼、卡號等等，payload由base64Url編碼，如果外洩等於明碼直接暴露

Custom Claims

意旨未在Registered Claims IANA Claims標準中，自訂義的Claims，而Custom Claims又分為兩格部分:

  • Public Claims
    通用的信息，例如用戶的姓名和電子郵件地址。這些信息可能在不同的應用程序中都會使用。

💡Public Claims需要注意名稱衝突，不同服務對於相同的Claims名稱可能有不同意義，舉例Role對於A服務是作者，而對於B服務是讀者，Claims名稱一致但意義不同，因此請用命名區隔避免混淆。

  • Private Claims
    私有聲明是用來傳遞應用程序專有的信息，
    例如員工ID或部門名稱。這些聲明不公開，僅用於你自己的應用程序內部。

Signature

簽名部分用於驗證消息在過程中沒有被篡改，並且可以使用指定的算法（例如HMAC SHA256）和一個Secret來生成簽名。

安全性

  • JWT是可公開的: 不要在JWT的Payload中存儲敏感信息，如密碼或信用卡信息。
  • 簽名密鑰的安全性: 如果使用對稱簽名算法，務必保證簽名密鑰的安全性。
  • 設置合理的過期時間（exp）: 防止令牌被長期濫用，或者需要更新訪問權限。
  • 使用HTTPS傳輸JWT: 防止中間人攻擊。

優點:

  • 無狀態: 伺服器不需要存儲JWT，方便與服務驗證交互。
  • 可擴展: JWT可以攜帶任意聲明，不僅僅限於身份驗證。

缺點:

  • Payload大小: JWT的Payload直接暴露，並且經過Base64編碼後可能會增大令牌的大小。
  • 不可撤銷: JWT一旦發出，在JWT過期前，無法撤銷它的使用，除非實作JWT管理系統。

💡在OAuth 2.0中JWT常被 ID Token、Access Token、 Refresh Token使用
