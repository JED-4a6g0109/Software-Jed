Docker 加速的技巧以及管理

發表於2025-01-27|更新於2025-12-14|DockerDevOps
|瀏覽量:

Docker在近幾十年內是目前大多企業最常使用的技術，在開發上Docker也是不可或缺的，而本篇文章將提出幾項如何讓Docker加快以及減少空間好管理得的技巧。

善用Docker Cache

在寫Dockerfile時，是否注意到每次在做Docker build image時總是特別慢，特別是在node install，或者 python poetry install，每次都浪費幾分鐘時間，其實在Dockerfile在內部就有Cache來幫你處理，直接舉個例子。

建置一個Python Django image正常情況下你會像下面這樣建立。

而每次開始docker build 會發現，docker從頭到尾會執行一次，而很多東西我們不需要每次都重新去安裝像是package，不需要因為改動商業邏輯的程式碼而每次都要重跑，而Dockerfile會依據你資料的變動來決定Cache，而這部分為Docker 的Layer，Docker 的每個指令（如 RUN、COPY 等）會生成一個 層（Layer），這些層構成了最終的映像。當執行 docker build 時，Docker 會嘗試從 Cache 中重用之前生成的層。因此你應該區隔開環境建置以及業務邏輯程式碼。

可以先Copy pyproject.toml然後再將專案其餘程式碼Copy近來，這樣每次在build就不會因為業務邏輯程式碼變動而重新執行poetry install。

執行時會看到CACHED代表已經HIT了。

不要分散多個Commands，請集中一起

當我們在 Dockerfile 中執行多個指令時，每一條指令都會創建一個新的層（Layer）。每個層會被 Docker 獨立存儲，並且每個層都會增加最終映像的大小。因此，為了減少映像的層數並提高效率，我們可以將多個指令合併成一條指令來減少層數。

你要避免這樣寫，前面提到Docker 的每個指令（如 RUN、COPY 等）會生成一個 層（Layer），所以這樣寫apk add 與pip install 就會生成兩層(Layer)，浪費空間以及效能。

盡量將Commands集中一起，寫在一起就只會生成一層(Layer)。

請依賴.env環境設置在你的docker compose上

Docker compose雖然能直接在上面設置帳號與密碼(ex: Redis、website manager etc..)，但請不要這樣做，在安全上不是最好方法且在切割環境時會衍伸很多問題，首先，創建一個 .env 文件，用來存儲環境變數。這個文件可以放在與 docker-compose.yml 同一目錄下，並且會被 Docker Compose 自動加載。

.env

在 docker-compose.yml 中，你可以使用 ${} 語法來引用 .env 文件中的環境變數。Docker Compose 會自動加載 .env 文件中的變數並替換它們。

視情況選用合適的image

docker以輕量化為其一特性，如果沒有善用這特性反而浪費你個多硬碟空間，你應該選擇合適的image以python 舉例，官方提供了不同python image。

使用.dockerignore

專案上可能有以下資料

  • .env
  • static image
  • test程式
  • git

如果不希望build 到image裡面可以使用.dockerignore來忽略，減少image的空間。
