Docker 基礎入門介紹

發表於2025-01-27|更新於2025-12-14|Docker
|瀏覽量:

Docker 是一種軟體平台，可讓您快速地建置、測試和部署應用程式。它通過將應用程式及其依賴環境打包(封裝)到一個輕量級、可攜帶的容器中，實現跨平台的一致性和高效性。

💡

在早期安裝一個服務在本地上(如Redis、Apache etc..)，如果要移植到另一個系統時，需要考量系統架構如X86或者ARM，且需要重新安裝或者執行Script shell，又或者不小心把環境弄髒或者弄壞又要重新清除又安裝，這些非常浪費時間又麻煩的事情。

Docker 術語

容器（Container）

顧名思義就跟docker logo上的貨櫃一樣概念，單一一個服務為一個貨櫃，而這些貨櫃在docker 稱為容器(Container)。容器是一個輕量級的、獨立的運行環境，包含應用程式及其所有的依賴項（如庫、工具、配置文件等）。它類似於虛擬機，但資源佔用更少，啟動速度更快。可以隔離多個服務之間互相影響。

鏡像（Image）

Image是Container的靜態模板，包含了運行應用程式所需的所有內容（如操作系統、應用程式代碼和依賴項）。Container是基於Image啟動的可運行實例。可以簡單使用DockerFile來建立客製Image

Docker 引擎（Docker Engine）

Docker 的核心組件，負責構建、運行和管理容器。

Docker Hub

一個雲端註冊表，用於存儲和分發 Docker 鏡像。你可以從 Docker Hub 上下載公共鏡像，也可以上傳自己的鏡像。

💡

其實不只有Docker Hub，像是AWS提供ECR也是可以存放在AWS雲服務上，這取決於自家架構運用在哪個服務商上。

Docker 與 VM

VM運行在Hypervisor（虛擬化管理程序）上，每個虛擬機都有自己的操作系統。而Docker 容器運行在共享的操作系統內核上，通過 Docker Engine實現應用隔離。

有用過VMware朋友都知道在一開始要安裝iso然後切割硬碟空間，然後運行上切太少vcpu或記憶體就會卡到不行，因為VM使用體積大，而每次啟動或關機都特別慢。

Docker 和 VM 的主要區別

特性 Docker（容器） 虛擬機（VM）
架構 共享主機的操作系統內核，通過容器運行應用程式。 每個虛擬機有完整的操作系統，運行在虛擬硬體上。
啟動速度 幾秒內啟動（輕量級）。 通常需要幾分鐘（因為要啟動整個操作系統）。
資源使用 更高效，因為不需要運行多個操作系統。 更耗資源，因為每個 VM 都需要自己的操作系統。
隔離性 容器之間隔離，但共享內核，隔離性稍弱。 完全隔離，每個 VM 都有自己的操作系統和內核。
存儲空間 體積小（幾十 MB 到幾百 MB）。 體積大（幾 GB 或更多）。
移植性 高移植性，容器可以在任何支持 Docker 的平台上運行。 移植性較低，虛擬機需要與虛擬化平台兼容。
性能 接近原生性能（共享主機內核）。 性能稍低，因為有虛擬化層的開銷。
適用場景 適合輕量級應用、微服務架構、快速部署與測試。 適合需要完整隔離的場景，如運行不同的操作系統。

到這邊你就會思單一服務還好，如果是多個服務像是一個網站會有資料庫、快取、伺服器、log分析等等多數服務，那要如何將他們包成image又或者綑綁呢? 不用擔心有docker compose可以幫你統一管理以及處理這些多服務!

Docker Compose

Docker Compose 是一個工具，專門用來管理和編排多個 Docker 容器的部署。當你的應用需要多個容器（例如一個 Web 伺服器、資料庫和緩存服務）時，Docker Compose 可以幫助你簡化這些容器的配置和啟動過程。可以看到下面圖中，章魚將每一個貨櫃綑綁一起，管理以及部屬。

docker-compose.yml 文件

  • Docker Compose 使用 YAML 文件來定義應用的服務、網絡和卷。
  • 你可以在這個文件中描述多個容器的配置，例如使用的映像、環境變數、端口映射等。

服務（Services）

每個服務對應一個容器，像是 Web 伺服器、資料庫等。

網絡（Networks）

Docker Compose 自動為服務之間創建網絡，讓它們能夠互相通信。

卷（Volumes）

用來持久化數據，確保容器重啟後數據不會丟失。

💡

因為Docker不是持久化的，每次重啟資料都不會保留，所以可以設置Volumes來持久化資料。

結論

docker 不僅加速了工程師在開發建置環境的時間以及測試，且能透過docker compose進行管理以及快速部屬，較於傳統需要一個個服務慢慢安裝來說便利許多，現今工程師如果還不會docker那要趕緊卷起來了!
