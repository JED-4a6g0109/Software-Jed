Array介紹

Array是最常被運用也做重要的資料結構，Array也提供了基本API像是建立、插入、更新、刪除、搜尋，除了直觀的API以外，經典演算法也是基於Array實作像是排序法，在資料結構與演算法中都是。

例子-生活中一週行程表（Weekly Schedule）

你的一週行程可以用陣列表示，每一天都是一個固定的位置：

Index 0 1 2 3 4 5 6
星期 日 一 二 三 四 五 六
活動 休息 上班 運動 會議 逛街 約會 聚會

Array在記憶體中是**連續儲存的，**以下舉個例子

例子-int 型別的陣列
假設 arr = [10, 20, 30, 40] 在記憶體中，arr[0] 起始位址為 1000（假設每個 int 佔 4 bytes），則記憶體位址如下：

索引 元素 記憶體位址（假設起始位址 1000）
0 10 1000
1 20 1004
2 30 1008
3 40 1012

而讀取可以透過Index來取值如:存取 arr[2]因此時間複雜度是O(1)

💡

連續儲存的在不同語言與儲存不同型態是不一樣的，例如python list就不是連續儲存的，而Java int Array是連續儲存，但Java String Array就不是連續儲存

除此之外Array在做新增與刪除特性是影響整個元素，每次插入時會將整個元素往後移，刪除也是會將整個元素往前移，因此刪除與插入在時間複雜度是O(n)

操作 靜態陣列 (array.array in Python)
存取（索引） O(1)
新增（尾部 append） O(1)
插入（中間） O(n)
刪除（中間） O(n)
刪除（尾部 pop） O(1)

除此之外python在定義list時，還會預留額外空間（over-allocation）。

外另python list是不連續記憶體空間儲存的。

Python 的 list不是連續記憶體存儲的，主要是為了支援以下兩個特性：

效能（動態擴展）

  • list 允許在不頻繁搬移元素的情況下增長容量，因此會有額外的預分配空間（over-allocation）。
  • 這讓 list.append() 操作通常是 O(1) 均攤時間複雜度，而不是 O(n)。

支援不同類型（多態性）

  • list 內部實際上是指標陣列（array of pointers），每個元素是對應物件的記憶體位址，而不是直接存數值。
  • 這讓 list 可以儲存不同類型的物件（例如 int、str、float、自定義物件混合存放）。
  • 但這也導致 list 佔用較多的記憶體，因為除了儲存元素本身，還要存儲指向元素的指標。
