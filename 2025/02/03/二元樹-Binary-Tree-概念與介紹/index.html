二元樹介紹

二元樹是一格樹狀的資料結構，因為結構像樹枝，而每個樹枝支點這邊稱為節點(node)，一個節點包含：

  • 數值或資料（Value）：存儲在節點中的資料。
  • 左指標（Left Pointer）：指向左子節點的指標。
  • 右指標（Right Pointer）：指向右子節點的指標。

左右子節點會比較數值本身來決定，比數值本身大放右子節點，反之小放左子節點，另外樹的對上方稱為根節點(root node)，而底部的節點沒有子節點所以稱為葉節點，

二元樹深度

我們可以將樹由上而下看，從縱軸 5 為第0層，3 和 8 為第1層，2、4、7 為第2層。這樣可以根據層級計算每層最多可以有多少個節點。對於一棵理想的二元樹，第 i 層最多可以有 2^i 個節點。因此，第 i 層最多可以容納的節點數是：

  • 第一層（i=0）：最多有 2^0 = 1 個節點。
  • 第二層（i=1）：最多有 2^1 = 2 個節點。
  • 第三層（i=2）：最多有 2^2 = 4 個節點。
  • 第四層（i=3）：最多有 2^3 = 8 個節點。

理想的情況下，每一層的節點數都不會超過 2^i。但是，實際情況下，樹的節點數量可能會有所不同，特別是在不平衡的二元樹中。

二元樹類型

二元搜尋樹

在二元搜尋樹中，搜尋節點的操作非常高效。每個節點的數值比其左子節點的數值要大，比其右子節點的數值要小，因此我們可以根據這一性質，從根節點開始搜尋，逐層縮小搜尋範圍。

特徵：

  • 如果根節點的數值等於目標數值，則找到目標。
  • 如果目標數值小於根節點的數值，則搜尋左子樹。
  • 如果目標數值大於根節點的數值，則搜尋右子樹。

重複這些步驟，直到找到目標節點或沒有更多節點可搜尋為止。

完全二元樹（Complete Binary Tree）

完全二元樹其中所有層（除了最後一層）都是滿的，且最後一層的節點都集中在最左邊。也就是說，最後一層的節點可以不完全，但必須從左到右排列，沒有空隙。

特徵：

  • 所有層除了最後一層，都是滿的。
  • 最後一層的節點都集中在最左邊。

例子：

A到G按順序由上而下，由左至右，符合完全二元樹

    D((D))

    F((F))
    G((G))

    A --> B
    A --> C
    B --> D
    B --> E
    C --> F
    C --> G

平衡二元樹（Balanced Binary Tree）

平衡二元樹每一個節點的左右子樹深度差不超過 1，這樣可以確保樹的深度是平衡的，從而避免因為深度過高而影響查詢效率。

💡

平衡二元樹，左右子樹的深度差（即 left height - right height）不能大於 1。

特徵：

  • 對於每一個節點，左右子樹的深度差的絕對值不會大於 1。
  • 確保樹的深度不會過大，保持查找、插入、刪除操作的效率。

平衡二元樹例子：


graph TD
    A((5))
    B((3))
    C((8))
    D((2))
    E((4))
    F((7))

    A --> B
    A --> C
    B --> D
    B --> E
    C --> F

根節點 5：

  • 左子樹高度：從 5 到 3，再到 2 或 4，最高為 2
  • 右子樹高度：從 5 到 8，再到 7，最高為 2
  • 高度差：2 - 2 = 0，這是平衡的。

節點 3：

  • 左子樹高度：只有節點 2，高度為 1
  • 右子樹高度：只有節點 4，高度為 1
  • 高度差：1 - 1 = 0，這是平衡的。

節點 8：

  • 左子樹高度：只有節點 7，高度為 1
  • 右子樹高度：沒有右子樹，高度為 0
  • 高度差：1 - 0 = 1，這是平衡的。

因此，這棵樹是平衡二元樹，因為每個節點的左右子樹深度差都不大於 1，符合平衡二元樹的條件。

判斷是否為平衡二元樹

根節點 5：

  • 左子樹高度：從 5 到 3，再到 2 或 4，最高為 2
  • 右子樹高度：從 5 到 8，再到 7，再到 6，最高為 3
  • 高度差：3 - 2 = 1，這是平衡的。

節點 3：

  • 左子樹高度：只有節點 2，高度為 1
  • 右子樹高度：只有節點 4，高度為 1
  • 高度差：1 - 1 = 0，這是平衡的。

節點 8：

  • 左子樹高度：沒有左子樹，高度為 0
  • 右子樹高度：從 8 到 7，再到 6，高度為 2
  • 高度差：2 - 0 = 2，這是不平衡的，因為高度差大於 1。

這棵樹是 不平衡的二元樹，因為節點 8 的左右子樹深度差超過了 1，不滿足平衡二元樹的條件。

完美二元樹（Perfect Binary Tree）

完美二元樹每一層的節點數量都達到最大數量，並且所有葉子節點都在同一層。換句話說，這是一個完全填滿的二元樹，沒有任何空隙。

💡

完美二元樹，每層的節點數量必定是 2^n，其中 n是層數。

特徵：

  • 每一層都是滿的。
  • 所有葉子節點都在同一層。

例子：

graph TD
    A((5))
    B((3))
    C((8))
    D((2))
    E((4))
    F((7))
    G((6))

    A --> B
    A --> C
    B --> D
    B --> E
    C --> F
    C --> G

二元樹彙整

樹的類型 規則

平衡二元樹 任意節點的左右子樹高度差不超過 1
搜尋二元樹（BST） 左子樹小於根，右子樹大於根

因為3節點的右子節點是空的，所以Index 4為null，這種儲存方式有個好處是可以找尋左右子節點索引以及父節點索引，可以透過公式計算

左子節點 (Left child)：

  • 左子節點的索引：2 * index + 1

右子節點 (Right child)：

  • 右子節點的索引：2 * index + 2

父節點 (Parent node)：

  • 父節點的索引：(index - 1) // 2(整數除法 //，保證返回的是父節點的整數索引)

例子

節點 3（索引 1）：

  • 左子節點：2 * 1 + 1 = 3，對應節點 2。
  • 右子節點：2 * 1 + 2 = 4，對應節點 4。
  • 父節點：(1 - 1) // 2 = 0，對應節點 5。

節點 2（索引 3）：

  • 左子節點：2 * 3 + 1 = 7，該位置是 null，表示無左子節點。
  • 右子節點：2 * 3 + 2 = 8，該位置是 null，表示無右子節點。
  • 父節點：(3 - 1) // 2 = 1，對應節點 3。

但這種方式有個缺點處理稀疏二元樹時，使用陣列來儲存會遇到一些挑戰。稀疏二元樹是指樹中有很多節點的子樹為空，這意味著樹的結構並不完全填滿每一層或每一個節點位置。在這種情況下，陣列的使用會變得不那麼高效，因為會有很多空位，浪費記憶體，且會造成空間的浪費。

搜尋二元樹應用與實驗

在旁行榜中，我們可以應用二元樹來實作找尋特定玩家分樹，並與陣列做效能比較，以下為請ChatGPT 生出來的 Python範例

二元搜尋樹（BST）：將會隨著玩家數量的增加，時間複雜度保持在 O(log n)，查詢速度將比陣列快得多。

陣列查詢：每次查詢的時間複雜度是 O(n)，即使陣列已經排序，查詢仍然需要遍歷陣列以找到指定的玩家。

特徵 二元樹 陣列
查詢速度 O(log n)，適合查詢有序數據 O(1)，隨機訪問快速
插入與刪除 O(log n)，適合頻繁插入與刪除 O(n)，插入與刪除需移動元素
內存效率 每個節點需儲存指標，內存較高 內存利用率較高，動態調整大小
簡單性與可讀性 需要更多代碼來實現操作，較複雜 實現簡單且直觀，易理解
應用場景 需要高效查詢、排序和動態更新的情境 隨機存取、遍歷或簡單插入/刪除的場景
性能差異 查詢、插入、刪除效率較高 查詢速度快，插入刪除較為低效

結論

二元樹不只有排行榜實作方式，還有許多像是PostgreSQL 的Index(B-Tree and B+ Tree)，以及圈圈叉叉Minimax 也有用到二元樹概念等等…，許多應用原理都是基於二元樹延伸概念，因此了解二元樹是演算法中最重要一環。
